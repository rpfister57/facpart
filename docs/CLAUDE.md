This file provides guidance to Claude Code (claude.ai/code) when working
with code in this repository.

## Project Overview

`facpart` is an installable R package providing tools for partitioning
2-dimensional point configurations (such as multidimensional-scaling
output) according to Facet Theory. It implements three partition types —
axial (parallel lines), radial (nested circles or ellipses), and angular
(wedge sectors) — plus ellipse-fitting utilities.

The git repository is rooted here at `facpart/`. This is a standalone
package; there is no dependency on any script files outside this
directory.

## Development Workflow

All development commands run from inside the `facpart/` directory (or
with the `facpart.Rproj` open in RStudio):

``` r

devtools::load_all()   # load all functions without installing (fastest for interactive dev)
devtools::document()   # regenerate man/ and NAMESPACE from roxygen2 comments
devtools::install()    # install locally
devtools::check()      # run R CMD check
devtools::build()      # build the source tarball (facpart_x.y.z.tar.gz)
```

Do not edit `NAMESPACE` by hand — it is auto-generated by roxygen2.

Run the test suite with:

``` r

devtools::test()                                          # all tests
testthat::test_file("tests/testthat/test-guttman.R")     # single file
```

Currently only `guttman.R` functions have formal tests
(`tests/testthat/test-guttman.R`). Ad-hoc testing for other functions is
done interactively via
[`devtools::load_all()`](https://devtools.r-lib.org/reference/load_all.html)
followed by manual calls.

## Internal Dependencies

`radial-ellipse.R` calls `.best_radius()` which is defined in
`radial-circle.R`. This cross-file dependency exists in two places: the
independent-center ellipse path (nesting constraint) and the fixed-shape
[`radialEllipses()`](https://rpfister57.github.io/facpart/reference/radialEllipses.md)
path (scale-factor scan). When loading the package this is invisible,
but if sourcing files interactively, source `radial-circle.R` first.

Private helpers use a `.` prefix (e.g., `.circle_pts`, `.in_ellipse`,
`.arc_mid`) and are tagged `#' @noRd`. They are internal to their file.

## Key Algorithms

### Ellipse types (`ellipse.R`)

- **`"maxdist"`** (default): covariance-based ellipse centered at the
  data centroid, scaled to the maximum Mahalanobis distance + 5% buffer.
- **`"bestfit"`**: minimises sum of squared normalised elliptic
  distances via BFGS; initial guess from covariance ellipse.
- **`"minbound"`**: exact minimax minimum-area enclosing ellipse via
  `ellipsoidhull()`.

[`ellipseInConfig()`](https://rpfister57.github.io/facpart/reference/ellipseInConfig.md)
supports `mid = "centroid"` (default) or `mid = "nullnull"` (forces
center at origin). The `angle` in all ellipse output is in **radians**.

### Axial partitions (`axial.R`)

- **[`axialLine()`](https://rpfister57.github.io/facpart/reference/axialLine.md)**:
  binary split. Classical LDA Bayes-rule boundary — perpendicular to LD1
  through the midpoint of class means. Requires exactly 2 group levels.
  Vertical-line guard: when `|w[2]| < 1e-10`, `slope = Inf` and
  `intercept` carries the x-position. Returns `predicted` (LDA class
  factor); the only function that does.
- **[`axialLines()`](https://rpfister57.github.io/facpart/reference/axialLines.md)**:
  k ≥ 2 groups, k−1 parallel lines. Joint search over **angle and
  cuts**: grid of `n_angles` angles in \[0, π) plus LDA’s LD1 direction;
  for each angle, exhaustive enumeration of C(n−1, k−1) cut placements.
  Ties broken by **largest minimum margin** (perpendicular distance from
  a cut line to the nearest point). Same vertical-line guard as
  [`axialLine()`](https://rpfister57.github.io/facpart/reference/axialLine.md).
  The cut search is **vectorised via cumulative per-group counts**
  (`cum[i+1, g]` = count of group g among the first i sorted points): a
  segment’s group counts are a vector subtraction
  `cum[hi+1,] - cum[lo+1,]`, and all C(n−1, k−1) candidate partitions
  are scored together with `pmax`/`pmin` — no per-combo R loop, no
  per-segment [`tabulate()`](https://rdrr.io/r/base/tabulate.html). This
  is ~17–23× faster than the previous per-combo loop for k ≥ 3 while
  returning bit-identical results (same enumeration and tie-break,
  `which.max` keeps the first-in-scan-order winner). Do not “simplify”
  this back to a nested loop over
  [`combn()`](https://rdrr.io/r/utils/combn.html) columns.
- **When to use which**: use
  [`axialLine()`](https://rpfister57.github.io/facpart/reference/axialLine.md)
  for the classical LDA classifier (optimal under Gaussian
  equal-covariance classes); use
  [`axialLines()`](https://rpfister57.github.io/facpart/reference/axialLines.md)
  for the empirically best parallel-line separator (can find a non-LD1
  direction that beats LDA on the training sample).

### Radial / nested circles (`radial-circle.R`)

- **[`radialCircle()`](https://rpfister57.github.io/facpart/reference/radialCircle.md)**
  /
  **[`radialCircles()`](https://rpfister57.github.io/facpart/reference/radialCircles.md)**:
  nested (inclusive) circles. Center optimised by **multi-start
  Nelder-Mead with `parscale`** set to the data range. Starts include
  the overall centroid, each group’s centroid, and (for
  [`radialCircles()`](https://rpfister57.github.io/facpart/reference/radialCircles.md))
  the previous circle’s center. Nesting constraint
  `r_s ≥ dist(center_s, center_{s−1}) + r_{s−1}` enforced via minimum
  radius during a brute-force radius scan (`.best_radius()`). `misclass`
  is a plain **integer**. Supplying both `cx, cy` to
  [`radialCircles()`](https://rpfister57.github.io/facpart/reference/radialCircles.md)
  makes all circles **concentric**.

### Radial / nested ellipses (`radial-ellipse.R`)

- **[`radialEllipse()`](https://rpfister57.github.io/facpart/reference/radialEllipse.md)**
  /
  **[`radialEllipses()`](https://rpfister57.github.io/facpart/reference/radialEllipses.md)**:
  elliptic analogue of the radial circle functions. Each ellipse has 5
  parameters `(cx, cy, a, b, angle)` optimised by Nelder-Mead with
  `parscale = c(x_range, y_range, max_range, max_range, pi)`.
  Multi-start includes the covariance ellipse of the inner groups plus
  an inflated copy of the previous ellipse. `misclass` is a **list
  `(n, indices)`** — count plus row indices. Angles are in radians
  internally; converted to degrees only for `draw.ellipse()`. When
  `ellipse` (a 5-vector) is supplied to
  [`radialEllipses()`](https://rpfister57.github.io/facpart/reference/radialEllipses.md),
  outer ellipses are uniform scalings sharing the fixed
  center/orientation/ratio; scale factors found by exact 1D scan via
  `.best_radius()`.
- **`misclass` type differs from circle functions**:
  [`radialEllipse()`](https://rpfister57.github.io/facpart/reference/radialEllipse.md)
  /
  [`radialEllipses()`](https://rpfister57.github.io/facpart/reference/radialEllipses.md)
  return `misclass` as a list `list(n, indices)`;
  [`radialCircle()`](https://rpfister57.github.io/facpart/reference/radialCircle.md)
  /
  [`radialCircles()`](https://rpfister57.github.io/facpart/reference/radialCircles.md)
  return `misclass` as a plain integer.

### Guttman monotonicity coefficient (`guttman.R`)

- **[`mu2()`](https://rpfister57.github.io/facpart/reference/mu2.md)**:
  Guttman’s mu2 for two vectors. Computed as
  `sum(dx * dy) / sum(|dx| * |dy|)` over all pairs (i, j), where
  `dx = x_i - x_j`. Returns values in \[−1, 1\]; `NaN` when all
  differences are zero. Uses pairwise NA deletion.
- **[`mu2df()`](https://rpfister57.github.io/facpart/reference/mu2df.md)**:
  Applies
  [`mu2()`](https://rpfister57.github.io/facpart/reference/mu2.md) to
  all column pairs of a data frame. Returns a full symmetric matrix
  (`as_dist = FALSE`, default) or a `dist` object of `sqrt(1 − mu2)`
  values (`as_dist = TRUE`). Unlike the partition functions, these have
  no plot side-effect.

### Angular partition (`angular.R`)

- **[`angularPartition()`](https://rpfister57.github.io/facpart/reference/angularPartition.md)**:
  k wedge sectors from a common center. Brute-force over
  `n × C(n−1, k−1)` candidate angle k-tuples; tie-broken by **largest
  minimum 2D distance** from any cut ray to the nearest point. Center is
  **optimised** when either `cx` or `cy` is `NULL` (default) via
  multi-start Nelder-Mead; starts are the overall centroid plus each
  group’s centroid. Ray length is computed from `par("usr")`.
  `angularPartition3()` (fixed k=3 reference) exists only in the
  companion script layer and is not part of this package.
- All exported drawing functions accept `add = TRUE` (default, draw on
  the existing plot) or `add = FALSE` (call
  [`plot()`](https://rdrr.io/r/graphics/plot.default.html) first).
  [`ellipseInConfig()`](https://rpfister57.github.io/facpart/reference/ellipseInConfig.md)
  labels no points when `add = FALSE`; all other functions also draw
  group labels via
  [`graphics::text()`](https://rdrr.io/r/graphics/text.html).

## Coding Conventions

- Use `_` not `.` for object names (e.g., `grp_int`, not `grp.int`).
- Indentation: 4 spaces.
- Do not use ggplot2 for plots; all graphics use base R.
- Never use function names as object names (e.g., avoid
  `levels <- levels(x)`).
- Private helpers are prefixed with `.` and tagged `#' @noRd`.
- Do not add top-level
  [`require()`](https://rdrr.io/r/base/library.html) calls — package
  imports handle dependencies.
- Every exported function returns `invisible(NULL)` when
  `output = FALSE`; this is a package-wide convention.
- [`ellipseInConfig()`](https://rpfister57.github.io/facpart/reference/ellipseInConfig.md)
  has a `labs` parameter that is currently unused but kept for backward
  compatibility — do not remove it.

## pkgdown / GitHub Pages

The package site is deployed to `https://rpfister57.github.io/facpart/`
via the workflow at `.github/workflows/pkgdown.yaml`. It triggers on
push to `main`, on published releases, and manually via
`workflow_dispatch`.

The workflow runs
[`pkgdown::build_site_github_pages()`](https://pkgdown.r-lib.org/reference/build_site_github_pages.html)
(output goes to `docs/`) and then deploys `docs/` to the `gh-pages`
branch using `JamesIves/github-pages-deploy-action`. Configuration is in
`_pkgdown.yml` (Bootstrap 5 template; URL matches `DESCRIPTION`’s `URL:`
field).

To rebuild the site locally:

``` r

pkgdown::build_site()
```

The `docs/` directory is excluded from the package tarball via
`.Rbuildignore`.
